# TEST-ASD — Prompt Contract

## Intent

Provide a pytest that uses Flask's `app.test_client()` to drive `/validatearb` and `/geniac` with the generated `sample_asd.docx`, asserting the endpoints produce correctly-shaped JSON. No browser or UI driver. Live agent calls are gated behind an env var.

## Linked issue

**#19** — Test suite — ASD validation via API only (no UI)

## Inputs

- `back-end/file_processing/data/sample_asd.docx` (generated by `build_sample_asd.py`).
- All Foundry env vars (for live runs).
- `DefaultAzureCredential`.

## Outputs

- `back-end/tests/test_asd_api.py`.

## Test cases

1. `test_validate_endpoint_accepts_docx` (`live_azure`) — POSTs the docx, asserts:
   - `response.status_code == 200`
   - response JSON is a `list`
   - at least one item has the keys `Type`, `Issue`, `Description`, `Principles`, `Mandatory`
   - at least one item's `Principles` field is non-empty (category tagging).
2. `test_geniac_endpoint_returns_iac_list` (`live_azure`) — POSTs the docx, asserts:
   - `response.status_code == 200`
   - response JSON is a `list` and length ≥ 1
   - each item is a string containing the literal substring `resource "aws_`.
3. `test_validate_endpoint_rejects_missing_file` — no `live_azure`; expects 400.
4. `test_validate_endpoint_dispatches_on_extension` — patches the parser; verifies `.docx` calls `parse_arb_docx`, `.pdf` calls `parse_arb`, `.txt` returns 400.

## Edge cases & clarifications

1. **Live tests without env vars** → skipped with `pytest.skip(reason="…")`.
2. **Agent slow / first-call cold start** → per-test timeout of 90 s; on timeout the test fails with the actual elapsed time logged.
3. **Agent returns malformed JSON** (legitimate edge case from MAF-ORCHESTRATOR #6) → test asserts fallback record shape exists rather than 500.
4. **Sample doc not yet generated** → fixture runs `build_sample_asd.py` on demand.
5. **CORS / Content-Type** — assert `Content-Type: application/json` on responses.
6. **Front-end compatibility** — golden snapshot of the response keys (no schema break).
7. **Flask debug mode** — disabled in test client to avoid double traceback noise.
8. **Idempotency** — running the test twice consecutively produces the same shape.
9. **Empty IaC sections** — `/geniac` returns `[]` not `null` if every IaC section is empty.
10. **Authorization errors** propagate as HTTP 500 with `error_code` field, not bare stack.

## Acceptance criteria

- [ ] `pytest back-end/tests/test_asd_api.py` passes locally when env vars are set.
- [ ] Same command **skips cleanly** with env vars unset (no failures).
- [ ] No browser library imported (assert via `grep`).
- [ ] All assertions use explicit `assert` statements, not magic comparisons.
